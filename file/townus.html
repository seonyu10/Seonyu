Undestanding

TownUs 2.0: A group buying platform for all organizations

There was a lot of plannings to change of UI/UX within flows to reflect business strategies. As the service was getting bigger, many new pages were added, and design renewal was frequent, requiring the establishment of an overall design system and improvements in UI design based on them.

Research & Problem

UI Renewals, how to design consistently ?

The first goal was to improve the customizing design tool to help make it fun and easy to users who use the tool even for the first time. Secondly, it was necessary to renovate the existing purchase room UI to reflect the group purchas process more intuitive. Through this UI renewal project, i started to think that Townus needed standardized components so as to spend less time focused on style and more time developing a better user experience.

I built a design system alongside the purchase room and the customizing tool UI redesign that documented implementation and design guidelines.

UI Renewal 01

The Purchase Room Renewals

It is simple to purchase products from general commerce. You can select a product and purchase it. However, purchases in TownUs is not possible without going through a 'purchase room'. It is a virtual space for customers to bring people and buy products at the lowest possible prices. Our goal is that users have to be sufficiently aware of the buying process without any help from the CS team through the renewal.

UI Renewal 02

T-Shirts Customizing Tool Renewals

One of the main features that Townus provide is “T-shirts Customizing Tool”. Users can choose a different color of the product or change the part of the product where they want to add clip arts, images or text. This features gives Townus a competitive edge over other services in that it gives users the opportunity to design custom products before they actually proceed group buying.

I worked closely with the commerce team to create a new variation to the existing customizing tool experience, which had not seen updates since its initial design 2 years ago.

Design System

The Establishment of Design System starting from the style guide

There was no written documentations of style guide before i started this project. The main goal of this design system was to make components which are resuable and code even without a designer.



Gaining a lot of hint from material design, i was able to design our own version of style guide document. The components like buttons, text fields, main UI layout were exported to Zeplin as components.

Takeaway

Cover the flows with the consistent UI Design

Because the UX of TownUs is very different from that of ordinary commerce, it was necessary to bring in the common UX of the existing commerce, but still make use of the features only offered by Townus.

Presenting a new concept, such as a ‘purchase room’ or a ‘customizing design tool' that were not familiar to customers through the UI was a challenge. Therefore, it was essential to make the UI itself explain its concept and function through well-structured UI design. Testing a lot of times of iterations and variations helped me to find the optimal design.
